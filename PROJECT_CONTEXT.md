# Евровидение у Морозовых 2026

## Что это за проект

Мобильный сайт для домашнего голосования по Евровидению с отдельными экранами под:

- голосование с телефона
- карточки артистов
- live-результаты на ТВ/проектор
- таблицу игроков комнаты
- админку хоста

Прод-домены:

- `https://morozoveuroparty.ru`
- `https://api.morozoveuroparty.ru`

Основная комната:

- `neon-arena`

## Текущий стек

- `frontend`: Next.js 14, TypeScript, Tailwind
- `backend`: Express + Socket.IO
- `deploy`: Docker Compose
- `proxy`: Caddy
- `auth`: session-based account flow
- `storage`: пока локальный state на сервере и локальные media/uploads

## Ключевые маршруты

- `/` — портал
- `/{roomSlug}` — хаб комнаты
- `/{roomSlug}/vote/{stageKey}` — голосование
- `/{roomSlug}/acts/{stageKey}` — карточки артистов
- `/{roomSlug}/live/{stageKey}` — экран результатов для проектора
- `/{roomSlug}/players/{boardKey}` — таблица игроков
- `/admin?room=neon-arena` — админка

## Этапы

- `semi1`
- `semi2`
- `final`

## Важные UX-принципы

- Всё проектируется mobile-first.
- Основной пользовательский сценарий — с телефона.
- ТВ/проектор идут по отдельным URL.
- Для артистов используется один общий visual language на всех этапах.
- Для `final` допускается отдельный контекстный блок про путь в финал или место в полуфинале.
- Для полуфиналов такой блок не нужен.
- В `vote/acts` рейтинг теперь хранится в двух слоях:
  - полный внутренний порядок для автосдвига
  - отдельный `placed acts` слой, чтобы UI честно показывал пустое состояние, пока пользователь сам ничего не выбрал

## Принятые продуктовые решения

- Название проекта в интерфейсе: `Евровидение у Морозовых 2026`
- Артисты и названия песен остаются в латинице.
- Названия стран локализуются.
- У карточек артистов:
  - сильный poster-блок сверху
  - ниже компактные chips
  - затем основной блок `Об исполнителе`
- Не дублируем один и тот же текст в нескольких визуальных пузырях.
- Заметки должны работать и на ПК, и на телефоне.

## Прод-окружение

Сервер:

- Ubuntu
- Docker Compose
- Caddy

Деплой идёт в:

- `/opt/eurovision_project`

Быстрые команды:

```bash
cd /opt/eurovision_project
docker compose ps
docker compose logs --tail=100
docker compose up -d --build frontend
docker compose up -d --build backend
docker compose up -d --build
```

## Важно по безопасности

- `deploy-secrets.local.env` не должен попадать в git
- секреты нужно держать только локально
- SMTP пока не обязателен для работы сайта, но нужен для реального восстановления пароля
